About this document

+============================================================+ | ANALOGTURING.JL // OFFICIAL MANUAL // v0.1.0 | | PHOSPHOR EDITION // READABLE BY HUMANS // NOT YOUR MOM | +============================================================+

This is the manual for AnalogTuring.jl, a Julia module that gives you a Turing-complete analog computational substrate on top of digital floating-point. Every value passing through the module gets a small bounded perturbation (default ±3%) and snaps back toward its baseline — unless you wrap it in crystalize, which makes it solid forever.

The manual follows the PHP Manual convention: how to use it comes first, then a complete API reference, then a deep dive on internals, then flowcharts, then design notes. Every concept is given twice — once in the GRUG caveman-for-dummies style, then again as an intermediate-academic explanation. Pick whichever lands.

grug make this thing because digital rocks too sharp. every time you add 2 + 3 you get 5 exact. real world not like that. real world rocks shake a little. your hand shake. your eye shake. brain shake. when brain pretend to be analog with too-sharp rocks, it break in funny ways. ties never break. equal-rocks all win at same time. cave too quiet. so this module say: every rock wiggle little bit, then come back home. unless you say "no, this rock solid, do not touch". that is whole idea. simple.

The library imposes a small zero-mean stochastic perturbation on every floating-point intermediate (the jitter) followed by an exponential pull toward a per-value baseline (the snap-back). Bounded by a global ratio ε ∈ [0, 0.10] with default 0.03, this perturbation preserves expectations to first order while breaking ties stochastically, which is exactly the property required to model competitive populations, fuzzy associative recall, ambient cortical activity, and any other dynamical regime where exact equality is a bug rather than a feature. The Crystalized{T} wrapper provides a typed opt-out, and the propagation rule — "result is exact iff every operand was crystalized" — keeps the system inductively safe: a single non-crystalized input is sufficient to re-randomize the entire downstream chain.

If you only read one section, read Getting started — you can build something useful in five minutes from there.

Getting started

1.1   Installation

The package is not yet in the Julia general registry. Install directly from the GitHub repo:

using Pkg
Pkg.add(url="https://github.com/grug-group420/analog-turing")

Or clone for local development:

git clone https://github.com/grug-group420/analog-turing
cd analog-turing
julia --project=. -e 'using Pkg; Pkg.instantiate(); Pkg.test()'

Requires Julia 1.9 or newer. Dependencies (Random, Statistics, Distributions) are pulled in automatically by Pkg.instantiate().

If Pkg.test() reports anything other than 2087 passed, file an issue with the failing test name and your Julia version. The tests are the contract.

1.2   Hello, jitter

Drop into the REPL and watch sums refuse to sit still:

using AnalogTuring

aadd(2.0, 3.0)
5.012883417629045

aadd(2.0, 3.0)
4.973159820774432

amul(4.0, 5.0)
19.94022183947113

That's the whole library in three lines. Every output is the true sum (or product) plus a uniform noise term in ±3% of its magnitude, then pulled back toward its baseline. Statistically the mean is exact; instantaneously, every read is different.

1.3   Opting out: crystalize

When you actually need exact arithmetic — loop counters, control thresholds, fixed constants — wrap the value:

aadd(crystalize(2.0), crystalize(3.0))
❄(5.0)        # exact, every time

aadd(crystalize(2.0), 3.0)
5.034827...   # MIXED inputs → still jittered

rule: all rocks solid → answer solid. one rock wiggle → answer wiggle too. wiggle is contagious. like flu. you cannot half-freeze water. either it ice, or it slosh. this is the only rule you need to remember.

Crystalization propagation follows an all-or-nothing rule rather than the more common first-class or dominant rule. Formally, for any n-ary operation f(x₁,…,xₙ), the result is wrapped in Crystalized if and only if ∀i. is_crystalized(xₙ) = true. This makes jitter the absorbing element of the (Crystalized, jittered) lattice and guarantees that a single un-frozen operand is sufficient to re-introduce stochasticity into the downstream computation. The choice prevents a class of subtle bugs where a partially-crystalized intermediate would otherwise propagate exactness through a chain that the user expected to be noisy.

1.4   Global configuration

Three knobs, all process-global and thread-safe:

get_jitter_ratio()         # → 0.03  (default)
set_jitter_ratio!(0.05)    # bounded to [0.0, 0.10]
disable_jitter!()           # ε := 0, ops become exact
enable_jitter!()            # restore the previous ratio
is_jitter_enabled()         # → Bool

Calling set_jitter_ratio! with a value outside [0.0, 0.10] throws ArgumentError. The upper cap is not negotiable — values above 10% break convergence in auntil_converged and produce non-monotone behavior in HopfieldCache recall.

2.1   Basic arithmetic

using AnalogTuring

# Plain ops -- always jittered.
aadd(1.0, 2.0)        # ≈ 3.0
asub(5.0, 2.0)        # ≈ 3.0
amul(4.0, 5.0)        # ≈ 20.0
adiv(10.0, 3.0)       # ≈ 3.333…   throws on /0
asqrt(2.0)             # ≈ 1.414     throws on negatives
aexp(1.0)              # ≈ ℯ          warns on overflow

# Comparisons -- crisp (Bool) and fuzzy (Float in [0,1]).
aeq(1.0, 1.000_001; tol=1e-3)   # true
afuzzy_eq(0.0, 0.5; tol=1.0)    # 0.7788  (Gaussian kernel)
afuzzy_lt(0.0, 10.0)             # 0.999…  (logistic)

# Aggregations.
xs = [1.0, 3.0, 2.0, 5.0, 4.0]
amin(xs)                  # ≈ 1.0
amax(xs)                  # ≈ 5.0
amean(xs)                 # ≈ 3.0
aclamp(7.0, 0.0, 5.0)     # ≈ 5.0  (jittered bound)
aselect(0.7, 10.0, 20.0)   # 70% chance of ≈10

2.2   Coin flips and weighted choice

# Macro form -- inline binary branch.
result = @coinflip 0.7 "heads" "tails"

# Function form.
coinflip(0.5)                  # Bool, fair-ish under jitter

# Pick one of N by weight.
weighted_coinflip([1.0, 2.0, 3.0])   # → 1, 2, or 3 (idx)

# Strict probability vector (must sum to 1.0 ± 1e-3).
categorical_coinflip([0.2, 0.5, 0.3])

# Lateral inhibition -- temperature controls softmax↔hardmax.
lateral_inhibition_coinflip([1.0, 2.0, 3.0]; sharpness=0.5)   # soft
lateral_inhibition_coinflip([1.0, 2.0, 3.0]; sharpness=5.0)   # hard, almost always 3

2.3   Branches, loops, recursion

# Crisp branch (deterministic on the predicate).
abranch(() -> x > 0, () -> "pos", () -> "neg")

# Fuzzy branch -- score in [0,1] is taken AS the probability.
afuzzy_branch(0.7, () -> "yes", () -> "no")

# Stochastic dispatch over many branches.
astochastic_branch([
    (0.6, () -> "A"),
    (0.3, () -> "B"),
    (0.1, () -> "C"),
])

# Bounded loop -- max_iterations ALWAYS enforced.
aloop(i -> println("step ", i), 10)

# Conditional loop with implicit cap.
counter = Ref(0)
awhile(() -> counter[] < 5, () -> counter[] += 1)

# Newton's method via fixed-point. Stable-for-N suppresses jitter false-pos.
sqrt2 = afixed_point(x -> 0.5 * (x + 2.0 / x), 1.0;
                      tol=1e-3, stable_for=5)

# Recursion with explicit max_depth.
fact = arecurse(crystalize(6.0)) do recurse, n
    n.value <= 1 ? crystalize(1.0) : amul(n, recurse(crystalize(n.value - 1)))
end
# → ❄(720.0)  -- exact because every input was crystalized

2.4   Memory: registers, accumulators, Hopfield

# Single-cell jittered register.
r = AnalogRegister(10.0; label="my_reg")
get_value(r)              # ≈ 10.0  (jittered each read)
set!(r, 42.0)
crystalize_register!(r)   # freeze; subsequent set! warns and no-ops

# Decaying accumulator.
acc = AnalogAccumulator(decay_rate=0.1)
accumulate!(acc, 10.0)
accumulate!(acc, 5.0)
value(acc)                 # ≈ 14.0  (10*0.9 + 5, plus jitter)
reset!(acc)

# Hopfield-style associative cache.
cache = HopfieldCache(capacity=64, crystalize_threshold=5.0)
store!(cache, [1.0, 0.0, 0.0], 42.0)
store!(cache, [0.0, 1.0, 0.0], 99.0)

# Fuzzy lookup -- close keys still match.
result = recall(cache, [0.95, 0.05, 0.0])
if result !== nothing
    val, sim = result
    println("got ", val, " with similarity ", sim)
end

# Top-K matches, sorted by similarity.
matches = recall_top_k(cache, [1.0, 0.0, 0.0], 3)

# Always-on ambient field -- nothing ever sits at exactly zero.
field = AmbientField(baseline=0.05, breadth=0.5)
samples = sample_field(field; n=100)

2.5   Strength fields and voting

# Build a competitive population of "beads".
pop = Population()
register!(pop, "rock_a"; strength=3.0)
register!(pop, "rock_b"; strength=8.0)
register!(pop, "rock_c"; strength=1.0)

# One firing round: who lit up this tick?
fired = fire!(pop)                       # Vector{String}

# Pick a single winner from those that fired.
winner = vote(pop, fired; sharpness=2.0)

# Or run repeated voting and take the dominant.
champ = winner_take_all(pop)

# Hebbian-style update.
bulk_decay!(pop; rate=0.01)              # everyone forgets a little
bulk_reinforce!(pop, ["rock_b"]; delta=0.5) # bump the winner

# Diagnostic.
println(alive_count(pop), " beads alive")

2.6   Error handling and retries

# Catch-and-fallback.
result = aguard(() -> asqrt(-1.0); on_error=:fallback, fallback=0.0)
# → 0.0  (DomainError swallowed by fallback)

# Retry with backoff -- recovers from transient failures (i.e. jittered into bad domain).
result = aretry(() -> asqrt(aadd(-0.0001, 0.0002));
                attempts=5, backoff_fn=k -> 0.0)

# Hard timeout (raises if work exceeds N seconds).
awith_timeout(() -> expensive_thing(), 2.0)

The library never returns sentinel error values silently. Every failure path throws an exception with a module-prefixed message (AnalogPrimitives.adiv: divide by zero, etc.) or emits an @warn for non-fatal anomalies. If you find a path that fails silently, that is a bug — please report it.

3.5   AnalogMemory [src]

TYPE AnalogRegister(initial::Real; label::String = "anon")

A single mutable cell. get_value(r) returns a freshly-jittered read; set!(r, x) writes; drift!(r; alpha, baseline_jitter) applies one snap-back step. Crystalize with crystalize_register!(r) — thereafter set! warns and is a no-op.

TYPE AnalogAccumulator(; decay_rate::Real = 0.05, initial::Real = 0.0, label)

Decaying running sum. accumulate!(acc, x) applies state ← jitter(state · (1-decay) + x); value(acc) returns a jittered read; reset!(acc; to=0.0) clears.

throws
ArgumentError on decay_rate ∉ [0, 1], non-finite initial, or non-finite input
TYPE HopfieldCache(; capacity::Integer = 256, crystalize_threshold::Real = 5.0, decay_rate::Real = 0.001)

Associative key-value store with strength-based eviction and crystalize-on-use. Lookup is a hybrid of cosine similarity and L2 distance — pure cosine fails on 1-D keys, so the implementation blends 50% cosine + 50% normalized L2 closeness.

  • store!(cache, key, val; initial_strength = 1.0) inserts or merges (via the _key_near predicate) into a similar entry, bumping its strength.
  • recall(cache, query; min_similarity = 0.5) → (value, sim) or nothing. Bumps strength on hit; auto-crystalizes when strength reaches the threshold.
  • recall_top_k(cache, query, k) → sorted matches.
  • Weakest non-crystalized entry is evicted when the cache is full. Crystalized entries are immune.
TYPE AmbientField(baseline::Real = 0.05, breadth::Real = 0.5)

Log-uniform sampler: sample_field(f; n) = baseline · exp(breadth · U(-1, 1)). Used to seed background activity that prevents zero-equilibria.

3.6   StrengthField [src]

TYPE StrengthBead(id::String; strength = 1.0, crystalized = false)

Single competitive unit. Fields: id, strength ∈ [0, STRENGTH_CAP], crystalized::Bool, fire_count::Int, is_grave::Bool. A bead is "grave" when its strength has fallen below STRENGTH_GRAVE_THRESHOLD (= 0.05) — grave beads refuse all bump_strength! calls.

FN bump_strength!(bead; delta::Real) · decay_strength!(bead; rate::Real = 0.05)

bump adds (jittered) delta, clamped to [FLOOR, CAP]. decay multiplies by (1 - rate) and may transition the bead into grave state (with a @warn). Crystalized beads ignore both.

FN activation_probability(bead) :: Float64

Returns ACTIVATION_FLOOR + (PEAK - FLOOR) · strength / STRENGTH_CAP = 0.1 + 0.8 · s/10. Even a strength-zero bead has a 10% ambient firing chance.

TYPE Population()

A keyed dictionary of beads with a per-population ReentrantLock. Methods:

  • register!(pop, bead) / register!(pop, id; strength, crystalized)
  • fire!(pop) → Vector{String} of ids that lit up this round (Bernoulli per bead via activation_probability; crystalized beads always fire)

  • winner_take_all(pop) → the most frequent winner across repeated firings

How it works

The previous section told you what the functions do. This section tells you why they do it that way. Each subsection has the GRUG version followed by an academic-tier explanation.

4.1   The jitter primitive

every number get little shake before grug use it. shake is small — like 3 hairs out of 100. shake is bounded so number never go crazy far. shake is fair — sometimes plus, sometimes minus, even-out over many uses. shake stop being fair if you ask for shake bigger than 10 hairs out of 100; that is why grug not let you. zero stay zero (no hair to wiggle on bald rock). magic-number minus-9999 also pass through clean — that is grug "do not touch this please" sign.

The core primitive is jitter_value(x; ratio) = x + δ where δ ~ U(-ratio·|x|, +ratio·|x|) — uniform on a magnitude-proportional support. Three properties matter:

  1. Zero-mean. E[δ] = 0, so any linear functional of jittered values converges to its exact counterpart in expectation. Sums, means, and dot products are unbiased estimators of their crisp versions.
  2. Bounded relative error. |jitter(x) − x| / |x| ≤ ratio, so multiplicative chains compose cleanly: an N-step chain has worst-case relative error (1+ratio)N − 1 ≈ N·ratio for small ratio.
  3. Fixed points preserved. jitter(0) = 0 and jitter(CRYSTALIZE_SENTINEL) = CRYSTALIZE_SENTINEL. The first is essential because zero-equilibria are common and stochastic perturbation around zero would create directional drift; the second provides a typed escape hatch usable from C-style FFI without the wrapper machinery.

The hard cap of ratio ≤ 0.10 is empirically chosen: above 10%, contraction mappings used by auntil_converged can lose monotone convergence, and the HopfieldCache recall-by-blended-similarity stops being a metric in the strict sense.

4.2   Snap-back as low-pass

after wiggle, number try to go home to its baseline. baseline is "where rock want to live". each tick, rock take a step toward home: 85% of way there. so rock that gets shoved by jitter does not stay shoved — it walks back. otherwise rock could random-walk far far away and forget where home is. with snap-back, rock always remembers.

snap_back(current, baseline; alpha) = alpha·baseline + (1-alpha)·current is a classic exponential-moving-average filter, equivalent to the discretization of the ODE &xacute;(t) = -κ·(x − baseline) with κ = -ln(1-alpha). Composed with the jitter step, it forms an Ornstein-Uhlenbeck-like process: xt+1 = (1−α)(xt + δt) + α·baseline. The stationary distribution has mean equal to baseline and variance proportional to ratio2/(2α), which means tightening α (faster snap) reduces effective noise without requiring a smaller jitter ratio. The optional baseline_jitter flag lets the baseline itself drift slowly — this is how the system models long-horizon adaptation without losing short-horizon stability.

4.3   Crystalize propagation

solid rock plus solid rock = solid rock. solid rock plus wiggly rock = wiggly rock. one wiggly rock pollute everything. like one drop of mud in clean water — whole bucket muddy now. this rule is "all-or-nothing". keep it simple. you want exact answer? freeze every input. one stray un-frozen rock and your nice exact chain becomes wiggly again. this is intentional. this is what stop sneaky bugs where you forgot one value was loose.

Crystalization defines a two-element lattice {loose, frozen} with loose as the absorbing element. The propagation rule is output = frozen iff ∀i: inputi = frozen. This is the conjunctive rule, in contrast to the disjunctive rule (any frozen input freezes the output) commonly seen in dimensional-analysis libraries.

The conjunctive choice is motivated by the contamination semantics we want: if any input has been re-randomized somewhere upstream, the user almost always wants the downstream value to stay re-randomized too — otherwise an accidental round-trip through a non-crystalized path would silently restore exactness, masking the upstream change. The cost of this rule is that to get a fully-exact computation the user must crystalize every input boundary, which is a small price for the discipline it forces.

The implementation lives in _maybe_jitter(result, all_crystalized). When all_crystalized = true, the result is wrapped in Crystalized{Float64} so that subsequent ops up the call stack can re-evaluate the predicate; otherwise a plain jittered Float64 is returned, ensuring downstream is_crystalized tests answer false and the contamination propagates further.

4.4   Ambient field guarantee

nothing in cave is ever fully off. always small wind. small heartbeat. small noise. why? because if everything truly zero, nothing can ever start. need tiny push to break stalemate. so every weight in voting has tiny floor. every fire chance has tiny floor. always 10% chance even weakest rock gets to speak. cave never silent. cave hum.

The library guarantees an activation floor in three places, all calibrated so that an exact-zero equilibrium is impossible by construction:

  • weighted_coinflip floors each weight at 1e-6, so a zero-weight entry still has a non-zero (if vanishingly small) selection probability.
  • jitter_threshold clamps coin-flip thresholds into [0.01, 0.99], so the threshold p = 0.0 still fires occasionally and p = 1.0 still misses occasionally.
  • activation_probability floors at ACTIVATION_FLOOR = 0.1: even a strength-zero bead fires 10% of rounds.

This is the ambient field in the GrugBot420 whitepaper sense (constant ρ17.5): a non-vanishing background activity that dissolves stalemates and prevents the population from collapsing into a fully-quiescent attractor. Mechanistically it is the analogue of cortical spontaneous firing.

4.5   Strength-biased activation

strong rock fire often. weak rock fire rare. but never never-fire and never always-fire. probability climb smooth from 10% (weakest) up to 90% (strongest). this is so loud rocks dominate but quiet rocks still have a voice. when many rocks fire, vote pick one by softmax-with-temperature. low temperature = soft vote (everyone has a chance). high temperature = hard vote (loudest always wins). same knob for both.

The activation function is the affine map p(s) = ACTIVATION_FLOOR + (ACTIVATION_PEAK − ACTIVATION_FLOOR) · s / STRENGTH_CAP, equivalently p(s) = 0.1 + 0.08·s for s ∈ [0, 10]. Voting is implemented via lateral_inhibition_coinflip(weights; sharpness) with weightsi = strengthi + ACTIVATION_FLOOR:

P(i wins) ∝ exp(sharpness · weighti)

This is a Boltzmann-Gibbs distribution with inverse temperature β = sharpness. The two limits are unified under one parameter:

  • sharpness → 0: uniform sampling (maximum entropy, "softmax" in the loose sense).
  • sharpness → ∞: argmax (zero entropy, "hardmax").

The grave-state mechanism (apoptosis at strength ≤ STRENGTH_GRAVE_THRESHOLD) implements use-it-or-lose-it pruning. Once grave, a bead refuses bumps — this prevents thrashing where a rarely-used pathway keeps getting reinforced just enough to never disappear. Crystalized beads are immune to both decay and grave-transition; this is how you pin "core" units that should never be culled.

4.6   Hopfield with hybrid lookup

cave-memory work like this: you walk in with a half-broken stick, cave find the closest stick it has seen, and tell you what came with that stick last time. closer match, stronger pull. but raw cosine-angle is bad — for one-number sticks, every stick points the same way, so cave thinks they all match. fix: also check how far apart sticks ARE, not just direction. blend the two. now sticks of length 1 and length 2 can be told apart. each time you ask cave for a stick, that memory gets a strength bump; ask enough times, cave freezes the memory permanent.

HopfieldCache is a content-addressable store with three innovations over the textbook Hopfield network:

  1. Hybrid similarity metric. Pure cosine fails on one-dimensional keys (all positive scalars are colinear, cosine = 1). The implementation blends 50% cosine + 50% normalized L2 closeness: 
    score = 0.5 · cos(key, query)
      + 0.5 · (1 − min(||key-query||₂ / max(||key||₂, ||query||₂), 1))
    This restores discriminability for low-dimensional keys without breaking the high-dimensional behavior.
  2. Strength-driven crystalization. Each recall bumps the matched entry's strength by 0.1 (jittered). Once strength reaches crystalize_threshold, the entry is crystalized: its value stops jittering on read, and it becomes immune to eviction. This is the system's mechanism for promoting frequently-recalled associations into long-term memory.
  3. Capacity-bound apoptosis. When the cache is full and a new entry needs to be stored, the weakest non-crystalized entry is evicted. If the cache is full of crystalized entries, the store is rejected with a @warn — the system tells you, loudly, rather than silently dropping the new entry or silently un-crystalizing an old one.

4.7   Turing-completeness argument

computer can compute anything if it has three things: way to choose (branch), way to repeat (loop), way to remember (state). grug have all three: abranch & friends for choose, aloop/awhile/arecurse for repeat, AnalogRegister & AnalogAccumulator for remember. so anything regular Julia can compute, grug can compute, just wiggly. this is the "turing-complete analog" claim.

The classical proof of Turing-completeness for any computational substrate requires three primitive capabilities: conditional dispatch, unbounded iteration, and persistent state. AnalogTuring.jl provides all three:

  • Dispatch: abranch (crisp), afuzzy_branch (probabilistic), astochastic_branch (N-way weighted). Any classical 2-way branch lifts to abranch directly.
  • Iteration: aloop, awhile, auntil_converged, and arecurse. Strictly speaking these are bounded by MAX_ITERATIONS_DEFAULT = 10,000 and max_depth = 64; this is intentional — an unbounded substrate is a denial-of-service substrate. Increase the caps if you need more, but the cap exists.
  • State: AnalogRegister (write-once-or-many cell), AnalogAccumulator (decaying integrator), HopfieldCache (associative store), Population (keyed dictionary of beads).

The arecurse + amul factorial integration test is the headline demonstration that all three compose: it is a recursive computation with conditional base case, multiplicative state composition, and the result is exact (720.0) when every input is crystalized — proving the analog substrate can simulate the digital one when asked, while remaining stochastic when not.

4.8   No-silent-failures discipline

thing break? grug must say so. loud. crash early. crash with name of function and what went wrong. never quietly return wrong number and pretend everything fine. that way, when something break, you know where. this is the most boring rule and the most important rule.

Every public function in the module follows the same error contract:

  • Input validation is the first thing in the body. ArgumentError, DomainError, DivideError, and DimensionMismatch are thrown with messages prefixed by Module.function: so a stack trace is sufficient to locate the failure.
  • Non-finite intermediates (NaN, ±Inf) trigger @warn rather than silent propagation; the warning includes the offending value and the operation that produced it.
  • Loop caps are enforced with explicit throw(ErrorException(...)); a non-converging fixed-point iteration never returns garbage, it raises.
  • Concurrency uses ReentrantLock per stateful structure (AnalogRegister, AnalogAccumulator, HopfieldCache, Population) plus a single _CONFIG_LOCK for global jitter knobs. No data race is possible through the public API.
  • Crystalization writes are policy-enforced: set! on a crystalized register issues @warn and is a no-op rather than silently mutating; the warning makes the semantic violation visible.

This costs a small amount of branch overhead at function entry but pays for itself the first time something downstream goes weird and the stack trace points you directly at the bad input.

Design constants

The numbers below are load-bearing. Each was chosen against a specific failure mode — changing any of them by more than a small fraction will cause one or more tests to flap or fail. They are exposed as exported constants so a user can adapt them, but treat any change as a deliberate design decision.

ConstantDefaultWhy this number
JITTER_RATIO_DEFAULT 0.03 3% feels analog (visible wiggle in the REPL) without breaking convergence in auntil_converged. Below 1% the jitter is invisible to the user; above 5% Newton's method needs many extra iterations.
JITTER_RATIO_MAX 0.10 Empirical ceiling. Above 10%, the multiplicative chain bound (1+r)N grows too fast for typical N; the Hopfield blended-similarity gate becomes non-monotonic.
CRYSTALIZE_SENTINEL -9999.0 Reserved exact-passthrough. Not a magic NaN because we want it to compare and arithmetize like a normal number through C-style code paths.
MAX_ITERATIONS_DEFAULT 10_000 Universal loop cap. Big enough for honest iterative numerics, small enough that an infinite loop is caught in well under a second on any reasonable hardware.
CONVERGENCE_TOL_DEFAULT 1e-6 About one part per million — below the floor of Float32 noise but above what jitter typically introduces at default ratio.
STRENGTH_FLOOR / STRENGTH_CAP 0.0 / 10.0 One decade of dynamic range. Wide enough that bumps and decays produce visible competition, narrow enough that activation_probability stays in [FLOOR, PEAK] with linear scaling.
STRENGTH_GRAVE_THRESHOLD 0.05 1/200 of the cap. Beads below this are functionally dead; setting it to 0 would let beads loiter at 1e-12 forever, setting it too high would prematurely kill recoverable beads.
ACTIVATION_FLOOR / PEAK 0.1 / 0.9 Symmetric 10%/90% bounds. Floor > 0 prevents fully-quiescent equilibria; peak < 1 prevents fully-deterministic firing — the population always has stochastic competition.
